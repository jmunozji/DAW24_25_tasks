    Cheatsheet para Instalación y Configuración de Servidores Web y Pila LAMP

    Configuración Inicial del Servidor

    1. Preparar el entorno:

    2. Configurar una máquina virtual (VM) o instancia en la nube con:

      • Sistema operativo: Debian (o similar basado en Linux).
      • Puertos abiertos según necesidades (SSH, HTTP, HTTPS, etc.).
      • Actualizar sistema operativo: 
        sudo apt update && sudo apt upgrade

    3. Configurar acceso SSH seguro:

    4. Crear o usar un par de claves existente.
    5. Configurar grupos de seguridad para limitar el acceso a los puertos necesarios.

    Instalación de Servidor Web

    1. Instalar Nginx o Apache:
    2. Para Nginx: 
      sudo apt install nginx
sudo systemctl start nginx

    3. Para Apache: 

      sudo apt install apache2
sudo systemctl start apache2

    4. Configurar directorios y permisos:

    5. Crear directorios para los sitios web: 
      sudo mkdir -p /var/www/sitio1

    6. Ajustar propietarios y permisos: 

      sudo chown -R www-data:www-data /var/www/sitio1

    7. Habilitar sitios virtuales:

    8. Crear un archivo de configuración para cada sitio en /etc/nginx/sites-available o /etc/apache2/sites-available.
    9. Activar sitios y reiniciar el servicio: 
      sudo ln -s /etc/nginx/sites-available/sitio1 /etc/nginx/sites-enabled/
sudo nginx -t
sudo systemctl restart nginx

    Habilitar Autenticación en el Servidor Web

    1. Instalar herramientas de autenticación: 

      sudo apt install apache2-utils

    2. Crear archivo de contraseñas: 

      sudo htpasswd -c /etc/nginx/.htpasswd usuario

    3. Configurar autenticación en el sitio:

    4. Modificar el archivo de configuración del sitio: 

      location /protected {
    auth_basic "Restricted Content";
    auth_basic_user_file /etc/nginx/.htpasswd;
}

    5. Reiniciar el servidor web: 

      sudo systemctl restart nginx

    Habilitar HTTPS

    1. Obtener un certificado SSL:

    2. Usar Let's Encrypt para certificados gratuitos: 

      sudo apt install certbot python3-certbot-nginx
sudo certbot --nginx -d example.com -d www.example.com

    3. Configurar HTTPS en el servidor web:

    4. Asegurarse de que el archivo de configuración incluye: 

      server {
    listen 443 ssl;
    ssl_certificate /etc/letsencrypt/live/example.com/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/example.com/privkey.pem;
}

    5. Validar y reiniciar: 

      sudo nginx -t
sudo systemctl restart nginx

    Instalación de la Pila LAMP

    1. Instalar componentes básicos:
    2. Apache: 
      sudo apt install apache2
    3. MariaDB: 
      sudo apt install mariadb-server
sudo mysql_secure_installation

    4. PHP: 

      sudo apt install php libapache2-mod-php php-mysql

    5. Configurar Apache con PHP:

    6. Editar /etc/apache2/sites-available/000-default.conf: 
      DocumentRoot /var/www/html
<Directory /var/www/html>
    AllowOverride All
</Directory>

    7. Reiniciar Apache: 

      sudo systemctl restart apache2

    8. Probar configuración:

    9. Crear un archivo info.php: 
      <?php
phpinfo();
?>
    10. Acceder desde el navegador: http://<server_ip>/info.php.

    Dockerización de Servicios

    1. Instalar Docker y Docker Compose: 

      sudo apt install docker.io
sudo apt install docker-compose

    2. Configurar servicios en docker-compose.yml: 

      version: '3.8'
services:
  web:
    image: php:apache
    volumes:
      - ./html:/var/www/html
    ports:
      - "8080:80"
  db:
    image: mariadb
    environment:
      MYSQL_ROOT_PASSWORD: rootpassword
      MYSQL_DATABASE: exampledb
      MYSQL_USER: exampleuser
      MYSQL_PASSWORD: examplepassword
    ports:
      - "3306:3306"

    3. Iniciar servicios: 

      docker-compose up -d

    4. Validar la configuración:

    5. Acceder al sitio en http://localhost:8080.
    6. Verificar la conexión a la base de datos desde PHP.

    Consideraciones Generales

    • Seguridad:
    • Configurar permisos adecuados para evitar accesos no autorizados.
    • Usar autenticación y cifrado HTTPS siempre que sea posible.
    • Pruebas:
    • Validar configuración tras cada cambio con nginx -t o apachectl configtest.
    • Automatización:
    • Utilizar scripts o herramientas como Ansible o Terraform para configurar entornos repetibles.
    • Backups:
    • Realizar copias de seguridad de configuraciones y bases de datos periódicamente.
